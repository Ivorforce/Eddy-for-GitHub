# Contributing to Eddy for GitHub

Contributions to Eddy are welcome.

[Bug reports](https://github.com/Ivorforce/Eddy-for-GitHub/issues) and fixes are most appreciated — small, self-contained changes that improve the app for everyone.

I prefer if new features are submitted as specs written by a human (as an [issue report](https://github.com/Ivorforce/Eddy-for-GitHub/issues)), rather than a pull request or LLM output. This way I have an easier time judging the feature and ensuring its implementation quality.

If you need a feature I am not adding, you can fork the project and design it on your own (or even use an AI to implement it).
